# PWA設定とスマホ最適化ガイド

## PWAアイコンの作成

PWAとして機能させるには、アイコン画像が必要です。以下のサイズのアイコンを作成してください：

1. `public/icon-192.png` - 192x192ピクセル
2. `public/icon-512.png` - 512x512ピクセル

### アイコンの作成方法

1. **オンラインツールを使用**
   - [PWA Asset Generator](https://github.com/onderceylan/pwa-asset-generator)
   - [Real Favicon Generator](https://realfavicongenerator.net/)

2. **手動で作成**
   - デザインツール（Figma、Photoshopなど）で192x192と512x512の画像を作成
   - アプリのロゴやアイコンを使用
   - PNG形式で保存

3. **一時的なアイコン（開発用）**
   - 単色の画像やプレースホルダーを使用可能
   - 本番環境では適切なアイコンに置き換えてください

## 概要情報が取得できない理由

Google Places APIの`editorial_summary`フィールドは、**すべての店舗で利用可能ではありません**。以下の理由が考えられます：

1. **データの有無**: すべての店舗に概要情報が登録されているわけではない
2. **地域による違い**: 地域によってデータの充実度が異なる
3. **店舗の種類**: チェーン店や人気店ほど概要情報が充実している傾向がある

### 対策

- 概要情報がない場合は、評価、営業時間、住所などの他の情報を表示
- 開発者コンソールでデバッグログを確認し、APIレスポンスを確認
- 可能な場合は、店舗のWebサイトから情報を補完
