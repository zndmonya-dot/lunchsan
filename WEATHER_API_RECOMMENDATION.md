# 個人開発の日本人向け 天気API おすすめ

## 🏆 おすすめ: Open-Meteo API（現在実装済み）

### なぜおすすめか
- ✅ **完全無料** - APIキー不要、登録不要
- ✅ **日本の天気情報に対応** - 日本の緯度経度で正確なデータを取得
- ✅ **実装が簡単** - 緯度経度を指定するだけで取得可能
- ✅ **制限なし** - 非商用利用なら無制限
- ✅ **個人開発に最適** - 商用利用も可能（要確認）

### 使い方
```typescript
// 緯度経度を指定するだけで取得可能
const response = await fetch(
  `https://api.open-meteo.com/v1/forecast?latitude=${lat}&longitude=${lng}&current=temperature_2m,weather_code&timezone=Asia/Tokyo`
)
```

### 取得できる情報
- 現在の気温
- 天気コード（晴れ、雨、雪など）
- 天気予報（最大7日間）
- 降水量、風速など

## その他の選択肢

### 1. 天気予報API（livedoor天気互換）

**特徴:**
- ✅ 気象庁のデータを使用（日本の天気情報に特化）
- ✅ 完全無料、APIキー不要
- ✅ 都道府県別の天気情報

**デメリット:**
- ❌ 緯度経度ではなく、都道府県IDや都市名で検索が必要
- ❌ 実装がやや複雑

**URL:** https://weather.tsukumijima.net/

### 2. OpenWeatherMap API

**特徴:**
- ✅ 無料プランあり（1分間に60リクエスト）
- ✅ 日本の天気情報に対応
- ✅ 実装が簡単

**デメリット:**
- ❌ APIキーの取得が必要
- ❌ 無料プランには制限あり

---

## 結論

**個人開発の日本人には、Open-Meteo APIが最適です。**

理由：
1. **APIキー不要** - すぐに使い始められる
2. **完全無料** - 個人開発でも安心
3. **日本の天気情報に対応** - 日本の緯度経度で正確なデータを取得
4. **実装が簡単** - 緯度経度を指定するだけ
5. **制限なし** - 非商用利用なら無制限

現在の実装では、Open-Meteo APIを優先使用し、失敗した場合にOpenWeatherMap APIをフォールバックとして使用しています。

**追加の設定は不要です。そのまま使用できます。**
